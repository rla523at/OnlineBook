# sources 학습 문서 작성 규칙

이 규칙은 `sources` 하위 학습 문서에 적용한다. 더 하위 디렉터리에 별도 `AGENTS.md`가 있으면 해당 디렉터리에서는 그 규칙도 함께 적용한다.

## 기준 독자

- 문서별 기준 독자는 해당 문서 주제를 처음 학습하는 독자로 잡는다.
- 문서가 특정 분야의 하위 주제를 다루면 그 분야의 초심자를 기준으로 한다.
- 기준 독자가 이미 알고 있다고 볼 수 있는 내용과 아닌 내용을 구분한다.
- 판단이 애매하면 핵심 흐름에 필요한 범위 안에서 최소 설명을 붙인다.

## 핵심 용어 첫 등장

- 핵심 용어가 처음 등장하면 현재 문서의 흐름을 이해할 수 있도록 정의, 역할, 필요한 이유 중 필요한 내용을 설명한다.
- 약어, 도구명, 프레임워크명은 첫 등장 시 범주를 함께 적는다.
- 자세한 설명이 다른 문서에 있으면 내부 링크를 연결한다.
- 링크만으로 설명을 대신하지 않는다. 링크를 열지 않아도 현재 문단의 핵심 흐름은 이해되어야 한다.

## 설명 범위

- 한 문서에 모든 배경지식을 넣지 않는다.
- 현재 문서의 주제를 이해하는 데 필요한 최소 배경은 현재 문서 안에 둔다.
- 세부 배경, 긴 절차, 별도 주제는 관련 문서로 분리하고 내부 링크로 연결한다.

## 용어 관계 설명

- `A = B` 형태로 설명하려면 A와 B가 항상 같은 의미인지 확인한다.
- 항상 같지 않으면 `A는 특정 조건에서 B와 대응한다`처럼 조건이 드러나는 표현을 사용한다.
- `같은 의미`, `동일하다`, `대체할 수 있다` 같은 표현은 실제로 의미와 사용 범위가 같은 경우에만 사용한다.
- 비슷한 용어가 함께 나오면 의미 범위와 사용 조건을 구분한다.
- 서로 다른 계층의 개념을 같은 의미처럼 쓰지 않는다.
- 같은 용어가 문서 안에서 다른 의미로 쓰이면 문맥별 의미를 명시한다.

## 표현 정확성

- `보통`, `거의`, `일반적으로`, `실무적으로` 같은 표현은 단독으로 사용하지 않는다.
- 위 표현을 사용할 때는 적용 조건, 예외, 판단 기준을 함께 적는다.
- 기준을 명확히 쓸 수 없다면 해당 내용을 단정하지 않는다.

## 예시 작성

- 예시는 설명하려는 개념, 관계, 조건과 정확히 맞아야 한다.
- 예외적인 상황을 일반적인 예시처럼 제시하지 않는다.
- 예시가 특정 조건에서만 성립한다면 그 조건을 예시 앞이나 뒤에 명시한다.
- 코드 예시는 핵심 동작을 이해하는 데 필요한 입력, 처리, 결과를 함께 설명한다.
- 명령어 예시는 어떤 상태를 바꾸는지 설명한다.

## 불확실한 내용 처리

- 확인되지 않은 내용은 사실처럼 단정하지 않는다.
- 불확실한 내용은 `확인 필요`, `추정`, `조건부 설명` 중 하나로 구분해 표시한다.
- 공식 문서, 코드, 표준 명세 등으로 확인할 수 있는 내용은 확인한 뒤 작성한다.

## 수정 후 검토

- 문서 수정 후 같은 용어가 문서 안에서 일관된 의미로 쓰였는지 확인한다.
- 같은 개념을 서로 다른 용어로 설명하지 않았는지 확인한다.
- 조건부 설명이 단정문처럼 쓰이지 않았는지 확인한다.
- 핵심 용어가 첫 등장 시 설명되었는지 확인한다.
- 내부 링크가 현재 문서의 설명을 대체하지 않는지 확인한다.
- 문서의 기준 독자가 한 줄 요약, 본문, 예시를 끊기지 않고 따라갈 수 있는지 확인한다.

## Jupyter Book 검증

- `sources` 하위 문서를 수정한 뒤에는 페이지가 의도대로 반영되고 Jupyter Book preview가 문제없이 만들어지는지 확인한다.
- 기본 검증 command는 `python jb_manager.py --test`다. 이 command는 `toc.yml`을 재생성한 뒤 `jupyter_book start`를 실행한다.
- 문서 파일을 추가, 삭제, 이동, rename한 경우에는 `toc.yml` 구조가 바뀌므로 `python jb_manager.py --test`를 다시 실행해서 TOC와 preview를 확인한다.
- 이미 `run_test` 또는 `jupyter_book start`로 preview server가 실행 중이고 본문만 수정한 경우에는 매번 server를 재시작하지 않아도 된다. 실행 중인 preview에서 수정 사항이 반영되는지 확인한다.
- 작업을 마칠 때는 `python jb_manager.py --test`를 실행했는지, 또는 이미 실행 중인 preview에서 확인했는지를 명시한다.
