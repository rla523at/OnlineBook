# Cloud

## 서버(server)란 무엇인가요?

서버는 “다른 사람(또는 다른 컴퓨터)의 요청을 받아서, 필요한 일을 처리해 주고 결과를 돌려주는 컴퓨터”다.

* 여러분의 노트북/휴대폰은 보통 “요청하는 쪽(클라이언트, client)”이다.
* 서버는 그 요청을 “24시간 받아서 처리하는 쪽”이다.

예시:

* 휴대폰에서 쇼핑몰 앱을 열어 상품 목록을 본다
  → 앱이 서버에 “상품 목록 주세요” 요청
  → 서버가 데이터베이스에서 상품 정보를 꺼내서 응답
  → 휴대폰 화면에 상품이 표시됨

즉, 서버는 보통 다음을 한다.

* 웹페이지나 API 제공(“요청 받으면 응답”)
* 파일 저장/전송
* 로그인, 결제 같은 비즈니스 로직 처리
* 데이터베이스와 연동해 데이터 읽기/쓰기

서버는 “역할(role)”의 개념이다.
어떤 컴퓨터든 남의 요청을 받아 처리하도록 구성하면 그 컴퓨터는 서버가 된다.

## “서버를 제공해주는 서비스”란 무엇인가요?
회사가 직접 데이터센터를 운영하는 경우를 생각하면 된다.

* 물리 서버(진짜 컴퓨터 장비)를 구매한다.
* 회사 서버실(또는 데이터센터)에 가져다 놓고 랙(rack)에 장착한다.
* 전원, 냉각(에어컨), 네트워크(스위치/라우터) 연결을 한다.
* 운영체제 설치, 보안 설정, 모니터링 구성, 장애 대응 체계를 만든다.
* 고장 나면 부품 교체/AS를 진행한다.
* 트래픽이 늘면 서버를 추가로 구매해서 설치한다(구매/배송/설치에 시간 소요).

즉, “서버를 구매해서 설치”는 단순히 컴퓨터 한 대를 사는 게 아니라,
서버가 안정적으로 24시간 돌아가도록 물리 환경과 운영까지 전부 직접 책임지는 방식이다.

* 예: AWS EC2처럼 서버(컴퓨터)를 빌려주는 서비스
* 예: 서버 설치/운영을 쉽게 해주는 관리형 서비스(Elastic Beanstalk 같은 것)

## “서버 한 대를 빌려 쓴다”는 무슨 뜻인가요? (클라우드 컴퓨팅)

AWS 같은 클라우드는 물리 서버를 AWS가 미리 잔뜩 준비해 두고, 그 자원을 나눠서 빌려주는 방식이다.

여기서 “빌린다”는 말은 현실의 렌탈처럼 “장비를 집으로 가져오는 것”이 아니다.
인터넷을 통해 “내 계정 안에 서버 1대가 생긴 것처럼” 사용할 권한을 받는다는 뜻이다.

### 빌리면 어떤 일이 생기나요?

* AWS 콘솔에서 버튼 몇 번으로 “가상 서버 1대(EC2 인스턴스)”를 만든다.
* 그 서버에는 CPU/메모리/디스크/네트워크가 할당된다.
* 여러분은 그 서버에 접속해서 프로그램을 설치하고 실행할 수 있다.
* 비용은 보통 “켜져 있는 시간(또는 사용량)”만큼 낸다.
* 필요 없으면 서버를 끄거나 삭제해서 비용을 줄일 수 있다.
* 트래픽이 늘면 서버를 더 만들기도 쉽다(물리 구매 없이).

비유:

* 서버 구매/설치 = 건물을 직접 사서 전기/수도/보안까지 모두 관리
* 서버 빌리기(클라우드) = 호텔 방을 필요할 때 빌려 쓰고, 전기/시설 관리는 호텔이 담당

## AWS EC2는 무엇인가요?

EC2는 AWS에서 “서버 한 대를 빌려 쓰는 서비스”다.
즉, “가상 서버(클라우드 서버) 1대”를 만드는 기능이 EC2다.

EC2를 쓰면 여러분은 다음을 할 수 있다.

* 서버를 만들고(OS 선택)
* 그 서버에 접속해서(SSH 등)
* 웹서버/애플리케이션을 설치하고
* 직접 운영한다(배포, 로그, 모니터링, 확장 등)

핵심: EC2는 “서버 자체”를 제공한다. 운영은 기본적으로 사용자가 책임진다.

## Elastic Beanstalk는 무엇이고, EC2와 어떤 관계인가요?

Elastic Beanstalk(EB)은 “애플리케이션을 쉽게 배포하고 운영하게 해주는 서비스”다.

초심자 관점에서 EB의 의미는 이렇다.

* EC2를 직접 만들고 설정하는 일을 줄여 준다.
* 로드 밸런서(트래픽 분산), 오토스케일링(서버 자동 증감), 모니터링 같은 운영 구성을 기본으로 묶어서 제공한다.
* 사용자는 주로 “코드를 올리고 배포”에 집중한다.

관계 정리:

* EC2 = 서버(가상 컴퓨터)
* Elastic Beanstalk = 그 서버(EC2)와 운영에 필요한 주변 설정들을 자동으로 구성/관리해 주는 상위 서비스
  (즉, EB는 내부적으로 EC2를 사용한다.)
